# 룰 업로드 샘플 파일

룰북 → **파일 업로드(⇪)** 에서 아래 파일을 올려 4단계 흐름(업로드 → 분석 → 검토·수정 → 적용)을 테스트할 수 있습니다.

| 파일 | 형식 | 설명 |
|---|---|---|
| `1_상세형.md` | 헤딩 + 필드 | 룰마다 카테고리·원칙·조건·판정·태그·근거를 모두 지정 (3건) |
| `2_체크리스트.txt` | 단순 불릿 | 점검 문장만 나열 → 나머지는 기본값, 검토 화면에서 채움 (6건) |
| `3_혼합_ISA포함.md` | 혼합 | 룰별 카테고리(공통·ISA) 다르게, 필드 일부만 (5건) |

## 입력 형식
- `## 제목` 또는 `- 항목` / `1. 항목` = 하나의 룰(점검 문장)
- 그 아래 `- 키: 값` = 필드

### 지원 키
| 키 | 값 |
|---|---|
| `카테고리` (룰셋) | `common` · `isa` · `irp` |
| `원칙` | 적합성원칙 · 적정성원칙 · 설명의무 · 부당권유행위 금지 · 소비자 권리 · 고령투자자 보호 · 판매절차(녹취·숙려) · 사후관리 · 품질 평가(감점) 등 |
| `고객조건` (조건) | 모든 고객 · 고령자 · 초고령자 |
| `판정` (판정성격) | 누락형 · 감점형 · 비계량형 |
| `태그` | 태그 코드 (예: `CONF_UND`, `CNST_REC`) — 쉼표로 여러 개 |
| `근거` | 근거 조항 ID (예: `KN_001`, `KN_003`) — 쉼표로 여러 개 |

> 태그·근거·원칙 값이 마스터에 없으면 자동으로 비워지고, **검토 화면에서 셀렉트로 채우거나 수정·삭제**할 수 있습니다.
> 필드를 안 쓰면 기본값(카테고리=공통, 고객조건=모든 고객, 판정=누락형)으로 지정됩니다.
